# Inicio de sesion — 2026-03-03

## Que se hizo en esta sesion

### Objetivo cumplido: Hardening de credenciales SQL (doc 0008)

Se implemento la **Alternativa 2: DPAPI ProtectedConfiguration** para que los `.exe.config` desplegados no tengan credenciales SQL en texto plano.

### Resumen de cambios

**1. ConfigProtector.exe** (NUEVO — `tools\ConfigProtector\`)
- Herramienta .NET 4.8 que cifra/descifra la seccion `connectionStrings` de un .config usando DPAPI Machine-level (`DataProtectionConfigurationProvider`).
- El instalador lo ejecuta automaticamente post-install para ambos configs.
- Soporte: `--decrypt` para desencriptar (no se usa en instalacion normal).
- Ya compilado por el usuario.

**2. Eliminacion de fallbacks hardcodeados** (CRITICO)
- `Common\Global.Datos\SQLServer.cs` — `CadenaporConfiguracion()` eliminada. Si falta ConnectionStrings["Local"], lanza `InvalidOperationException` con mensaje claro. Usa `SqlConnectionStringBuilder` para parsear `InitialCatalog`.
- `Common\Global.Datos\SQLAccess.cs` — misma limpieza. Ahora lee "Local" (antes leia "ConTocayAnda").
- `Common\Global.DigitalPersona\EnrollmentForm.cs` — connection string hardcodeada `NOTE\SQL2014/sa/Soporte1` (linea 97) reemplazada con lectura de ConnectionStrings["Local"].

**3. Limpieza de credenciales en codigo fuente**
- Comentarios con connection strings eliminados de: `SQLServer.cs`, `SQLAccess.cs`, `ConexionBaseDatos.cs`, `MainForm.cs`.
- `app.config` de desarrollo (Fichador y Administrador): eliminadas todas las connection strings extra (ksk, Remoto, ConTocayAnda, PRD01, Azure). Solo queda "Local" con `Integrated Security=True`.

**4. Instalador actualizado** (`InstaladorUnificado\setup.iss`)
- `#define SourceProtector` apunta a `tools\ConfigProtector\bin\Release\net48`.
- ConfigProtector.exe incluido en [Files] → se instala en `{app}\tools\`.
- Procedimiento `ProtegerConfigs`: ejecuta ConfigProtector para cada .config post-install (paso 7).
- Procedimiento `AplicarACL`: `icacls` con SIDs para Admins=Full, SYSTEM=Full, Users=RX (paso 8).

**5. Compilacion exitosa**
- Fichador: `TEntradaSalida.exe` compilado en Release ✅
- Administrador: `Acceso.exe` compilado en Release ✅
- Common (4 DLLs): compilado ✅
- MSBuild usado: `C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe`
